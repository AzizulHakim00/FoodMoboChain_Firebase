# v1.3 architecture

The Android client reads and writes Firebase Realtime Database through a centralized service. Store, food, cart, order, address, support, rental, training and news records are protected independently by Security Rules.

Checkout reads the official food catalogue, groups cart lines by vendor and store, then writes one canonical order per store. The cart is cleared in the same multi-location update after successful validation.

The starter marketplace is generated by the administrator from known sample IDs, allowing repeatable refresh without touching real user-generated records.
